# Expo 项目开发文档

## 1. 项目概览

本项目是一个基于 Expo 框架创建的 React Native 移动应用，支持 Android、iOS 和 Web 三端运行。项目使用 Expo SDK 54 最新版本，采用文件系统路由（Expo Router）架构，集成了现代化的开发工具和最佳实践。

- **多平台支持**：一次开发，多平台运行（Android、iOS、Web）
- **现代化技术栈**：React 19.1.0、React Native 0.81.5、TypeScript
- **路由系统**：Expo Router 实现文件系统路由
- **主题支持**：内置深色模式和浅色模式
- **动画效果**：集成 react-native-reanimated 实现流畅动画
- **性能优化**：使用 expo-image 优化图片加载

## 3. 项目结构

```
e:\develop\expo\demo\my-app\
├── app/                 # 应用路由和页面
│   ├── (tabs)/          # 标签导航相关页面
│   │   ├── _layout.tsx  # 标签导航布局
│   │   ├── index.tsx    # 首页
│   │   └── explore.tsx  # 探索页
│   ├── _layout.tsx      # 根导航布局
│   └── modal.tsx        # 模态框页面
├── assets/              # 静态资源
│   └── images/          # 图片资源
├── components/          # 可复用组件
│   ├── ui/              # UI 组件
│   ├── hello-wave.tsx   # 波浪动画组件
│   ├── parallax-scroll-view.tsx  # 视差滚动组件
│   ├── themed-text.tsx  # 主题文本组件
│   └── themed-view.tsx  # 主题视图组件
├── constants/           # 常量定义
│   └── theme.ts         # 主题配置
├── hooks/               # 自定义钩子
│   ├── use-color-scheme.ts  # 颜色方案钩子
│   └── use-theme-color.ts   # 主题颜色钩子
├── scripts/             # 脚本文件
│   └── reset-project.js # 重置项目脚本
├── package.json         # 项目配置和依赖
└── tsconfig.json        # TypeScript 配置
```

### 3.1 目录说明

- **app/**：应用的路由和页面，采用文件系统路由，每个文件对应一个页面
- **assets/**：静态资源文件夹，包含图片等资源
- **components/**：可复用组件，按功能分类
- **constants/**：常量定义，如主题配置
- **hooks/**：自定义 React 钩子，用于逻辑复用
- **scripts/**：工具脚本，如项目重置脚本

## 4. 核心功能模块

### 4.1 导航系统

- **根导航** (`app/_layout.tsx`)：使用 Stack 导航器，管理标签导航和模态框
- **标签导航** (`app/(tabs)/_layout.tsx`)：使用 Tabs 导航器，包含首页和探索页
- **页面路由**：通过文件系统自动生成路由，无需手动配置
- **模态框**：支持从页面导航到模态框页面

### 4.2 首页 (`app/(tabs)/index.tsx`)

- **视差滚动效果**：使用 ParallaxScrollView 实现头部图片视差
- **欢迎界面**：包含欢迎文本和波浪动画
- **快速开始指南**：三步入门指南（尝试编辑、探索功能、重置项目）
- **链接与菜单**：展示 Link 组件的使用，支持触发操作和分享

### 4.3 探索页 (`app/(tabs)/explore.tsx`)

- **功能介绍**：详细展示项目包含的核心功能
- **可折叠组件**：使用 Collapsible 组件分类展示信息
- **平台特定内容**：根据运行平台显示不同内容
- **外部链接**：提供相关文档的链接，便于学习

### 4.4 主题系统

- **颜色方案**：支持浅色模式和深色模式
- **主题组件**：ThemedText 和 ThemedView 等主题感知组件
- **自定义钩子**：useColorScheme 用于获取当前颜色方案，useThemeColor 用于获取主题颜色

### 4.5 动画效果

- **波浪动画**：HelloWave 组件使用 react-native-reanimated 实现
- **视差滚动**：ParallaxScrollView 组件实现滚动时的视差效果
- **触觉反馈**：HapticTab 组件提供标签切换时的触觉反馈

## 5. 开发流程

### 5.1 环境设置

1. **安装 Node.js**：确保安装了 Node.js 16.0 或更高版本
2. **安装 Expo CLI**：`npm install -g expo-cli`
3. **克隆项目**：`git clone <仓库地址>`
4. **安装依赖**：`cd my-app && npm install`

### 5.2 运行项目

- **启动开发服务器**：`npm start`
- **在 iOS 模拟器运行**：按 `i`
- **在 Android 模拟器运行**：按 `a`
- **在 Web 浏览器运行**：按 `w`
- **在实体设备运行**：使用 Expo Go 应用扫描终端中的二维码

### 5.3 项目重置

当需要清除示例代码并获得干净的项目结构时，运行：
```bash
npm run reset-project
```
此命令会将当前 `app` 目录移动到 `app-example` 并创建新的空 `app` 目录。

### 5.4 代码规范

- **TypeScript**：使用 TypeScript 确保类型安全
- **ESLint**：运行 `npm run lint` 检查代码规范
- **代码风格**：遵循 Expo 官方推荐的代码风格

## 6. 最佳实践

### 6.1 路由与导航

- 使用文件系统路由，将页面按功能组织到对应目录
- 对于需要标签导航的页面，使用 `(tabs)` 目录分组
- 使用 `Link` 组件进行页面跳转，支持传递参数

### 6.2 组件开发

- 将可复用组件放置在 `components` 目录
- 对于与主题相关的组件，使用 `useThemeColor` 钩子
- 对于需要动画的组件，优先使用 react-native-reanimated

### 6.3 状态管理

- 对于简单状态，使用 React 的 `useState` 和 `useEffect`
- 对于复杂状态，考虑使用 Context API 或状态管理库（如 Redux）

### 6.4 性能优化

- 使用 `expo-image` 优化图片加载和缓存
- 避免在渲染过程中执行昂贵操作
- 合理使用 `useMemo` 和 `useCallback` 缓存计算结果和函数

### 6.5 多平台适配

- 使用 `Platform.select` 处理平台特定代码
- 测试应用在不同平台和设备上的表现
- 确保 UI 在不同屏幕尺寸上的适配

## 7. 常见问题与解决方案

### 7.1 依赖安装失败

- 确保网络连接正常
- 尝试使用 `npm install --legacy-peer-deps`
- 清理缓存：`npm cache clean --force` 后重新安装

### 7.2 运行时错误

- 检查终端输出的错误信息
- 确保所有依赖版本兼容
- 对于 Web 平台，检查是否有平台特定 API 未做兼容处理

### 7.3 打包构建失败

- 确保所有依赖已正确安装
- 检查 `app.json` 配置是否正确
- 对于 Web 平台，确保所有依赖支持 Web 环境

## 8. 部署指南

### 8.1 构建应用

- **构建 Android APK**：`eas build --platform android`
- **构建 iOS IPA**：`eas build --platform ios`
- **构建 Web 版本**：`npx expo export --platform web`

### 8.2 发布应用

- **使用 EAS Submit**：`eas submit --platform android` 或 `eas submit --platform ios`
- **Web 部署**：将构建产物部署到静态网站托管服务（如 Vercel、Netlify）
